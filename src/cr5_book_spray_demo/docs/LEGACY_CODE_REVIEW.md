# 旧代码复用建议

旧 `arm_pointcloud_processor` 已经包含：

- RGB、对齐深度和 CameraInfo 同步；
- 深度反投影；
- 相机点云变换到 `base_link`；
- YOLO TensorRT 推理；
- 目标点云发布。

本包没有直接复制旧节点，原因是：

1. 旧逻辑把整个检测框内的深度都当作目标，桌面和背景容易混入；
2. 点云用于持续累积，不是稳定目标锁定；
3. TF 失败时回退到“最新变换”，不适合实机轨迹的严格时序；
4. 旧模型类别未必是书本；
5. 识别、点云、累计和 MoveIt 耦合在同一节点，难以设置清晰安全门。

建议复用内容：

- D455 topic 命名；
- 当前 TF frame 命名；
- 本地 TensorRT/ONNX 工具链；
- 已验证的 camera 固定外参；
- 旧 YOLO 模型部署方式。

第二阶段升级方式：

```text
训练 book 实例分割模型
→ 输出精确 mask
→ 用 mask 替换本包的绿色背景矩形 mask
→ 后续深度平面、书本坐标、锁定和 MoveIt 保持不变
```
