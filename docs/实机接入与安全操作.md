# 实机接入与安全操作

## 网络拓扑

```
Ubuntu 笔记本
├── Wi-Fi ────────── 互联网 / GitHub（默认路由）
└── 有线网口 ──────── <CR5_CONTROL_BOX_IP> ── CR5 控制柜
                      （不设网关、不设 DNS）
```

有线网口仅用于 DOBOT CR5 控制柜通信，所有互联网流量通过 Wi-Fi。

## 实机启动

### 1. 机械臂上电

确认控制柜电源接通，急停按钮未按下。

### 2. 启动 ROS 驱动

```bash
cd ~/cr5_ros1_ws
source devel/setup.bash
roslaunch dobot_bringup bringup.launch
```

驱动包包含:
- TCP 通信 (`tcp_socket.cpp`)
- CR5 运动控制 (`cr5_robot.cpp`)
- 状态消息 (`RobotStatus.msg`, `ToolVectorActual.msg`)
- 控制服务 (MovJ, MovL, ServoJ 等)

### 3. 启动 MoveIt (如需要)

```bash
roslaunch cr5_moveit demo.launch
```

## D455 相机实机接入

相机通过 USB 3.0 连接，使用 RealSense ROS 驱动：

```bash
roslaunch realsense2_camera rs_camera.launch \
  serial_no:=<D455_SERIAL>
```

每次实机操作使用本地配置文件 `config/local.yaml` 指定相机序列号。

## 实机 TF

```
world → base_link → Link1..Link6 → tool0
```

实机 base_link 在控制柜坐标系中定义，与仿真不同。

## 安全规则

### 必须遵守

1. **急停随时可用**: 操作前确认急停按钮位置和功能
2. **低速验证**: 首次运动使用低速 (speed_factor ≤ 20)
3. **干运行**: 仿真中验证所有轨迹后再上实机
4. **工作区清空**: 机械臂运动范围内不得有人或障碍物
5. **程序审查**: 自动生成轨迹需人工审核后再执行

### 禁止

- 禁止在未测试情况下直接执行自动喷涂程序
- 禁止绕过急停和安全联锁
- 禁止在无防护情况下高速运行
- 禁止未经验证的远程控制

## 仿真 → 实机切换检查清单

1. 确认仿真中所有轨迹已通过干运行验证
2. 确认 MoveIt 规划无碰撞
3. 确认工具坐标系配置正确
4. 实机低速执行一遍完整路径
5. 确认所有安全条件满足
6. 逐步提升至工作速度

## 本地配置

实机 IP、相机序列号等敏感信息存放在 `config/local.yaml`（不提交 Git）。

模板文件: `config/local.example.yaml`

```yaml
workspace: "<WORKSPACE>"

robot:
  control_box_ip: "<CR5_CONTROL_BOX_IP>"

cameras:
  cam_front_left:
    serial: "<D455_SERIAL_FRONT_LEFT>"
  cam_front_right:
    serial: "<D455_SERIAL_FRONT_RIGHT>"
  cam_rear:
    serial: "<D455_SERIAL_REAR>"
```
