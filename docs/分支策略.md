# 分支策略

## 永久分支

| 分支 | 职责 |
|------|------|
| `main` | 已验证、可构建、可运行的稳定工程基线 |
| `develop/multi-camera-calibration` | 三相机外参与三维重建开发 |

## 短期任务分支

命名规范:
- `feature/<主题>` — 新功能
- `fix/<主题>` — 修复
- `chore/<主题>` — 工程整理

完成后必须合并到 develop 或 main，然后删除远端分支。

## 合并流程

```bash
# 从 develop 创建功能分支
git checkout develop/multi-camera-calibration
git checkout -b feature/my-task

# 开发、提交、推送
git push -u origin feature/my-task

# 完成后的合并 (在 GitHub 上创建 PR)
# 或本地合并:
git checkout develop/multi-camera-calibration
git merge --no-ff feature/my-task
git push origin develop/multi-camera-calibration

# 删除功能分支
git push origin --delete feature/my-task
```

## 禁止

- 直接在 `main` 上做大改动
- `git push --force origin main` (除非人工明确授权)
- 长期存在的 `fix/...-v1` 风格分支
- 创建 `test/`、`temp/`、`backup/` 等临时远端分支
