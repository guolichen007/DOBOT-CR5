# 系统架构

## 包结构

```
cr5_ros1_ws/src/
├── cr5_spray_sim/          # 仿真与标定主包
│   ├── launch/             # roslaunch 文件
│   ├── scripts/            # Python/shell 脚本
│   ├── config/             # YAML 配置
│   ├── models/             # Gazebo 模型
│   ├── urdf/               # 机器人 URDF/Xacro
│   ├── worlds/             # Gazebo 世界文件
│   └── src/cr5_spray_sim/  # Python 包 (aruco_compat)
├── cr5_spray_perception/   # Open3D 运行时
├── cr5_spray_planning/     # MoveIt 规划 (预留)
└── dobot_bringup/          # DOBOT 驱动
```

## TF 树

```
world → dummy_link → base_link → Link1..Link6 → spray_nozzle_frame
world → object_frame → calibration_target_frame
  ├── calibration_target_front_frame
  ├── calibration_target_left_frame
  ├── calibration_target_right_frame
  ├── calibration_target_top_frame
  └── calibration_target_back_frame
```

object_frame 由 `publish_object_pose.py` 根据 Gazebo 模型位姿发布。
标定目标内部 frame 由 `publish_calibration_target_frames.py` 通过 StaticTransformBroadcaster 发布。

## 三相机配置

| 相机 | 位置 | 光学 Frame |
|------|------|-----------|
| cam_front_left | 前左立柱 | cam_front_left_color_optical_frame |
| cam_front_right | 前右立柱 | cam_front_right_color_optical_frame |
| cam_rear | 后方立柱 | cam_rear_color_optical_frame |

相机位姿由 `spawn_fixed_cameras.py` 从 `config/simulation_scene.yaml` 读取，
使用内联 look-at 计算确定朝向，确保光学 +Z 轴对准标定目标。

## 启动流程

1. **A0**: 审计旧仿真进程 (`audit_sim_processes.py`)
2. **A1/A2**: 端口选择 + roscore 启动
3. **B**: roslaunch `spray_simulation.launch` (paused=true)
4. **C1**: 等待模型生成 (`wait_scene_models.py`)
5. **C2**: 绝对几何验证 (`check_scene_geometry.py`)
6. **D1**: unpause → 时钟验证 → controller 顺序启动
7. **D2**: CR5 运动链 + 相机话题 + 信号检查
8. **E**: 就绪 — 可进行相机采集和标定

## 会话环境

启动脚本在 roscore 就绪后写入 `/tmp/cr5_spray_simulation.env`。
第二终端 source 后可直接使用所有 ROS 工具，无需手动设置环境变量。
