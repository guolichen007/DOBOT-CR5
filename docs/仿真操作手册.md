# 仿真操作手册

## 构建

```bash
cd ~/cr5_ros1_ws
catkin_make
source devel/setup.bash
```

## 启动仿真

```bash
# GUI 模式 (推荐调试用)
bash src/cr5_spray_sim/scripts/run_simulation.sh \
  --gui --object=calibration_target --profile=quality

# Headless 模式 (批量测试)
bash src/cr5_spray_sim/scripts/run_simulation.sh \
  --headless --object=calibration_target --profile=quality --strict
```

## 命令行参数

| 参数 | 可选值 | 说明 |
|------|--------|------|
| --gui | flag | 显示 Gazebo GUI |
| --headless | flag | 无头模式 (默认) |
| --object | calibration_target, motor_housing_cylinder, rectangular_housing | 工作对象类型 |
| --profile | vm, quality | 相机分辨率 profile |
| --strict | flag | 任何检查失败立即退出 |
| --isolated | flag | 独立模式 (随机端口) |
| --no-spray-sim | flag | 不启动喷涂控制 |
| --verbose | flag | 打印 roslaunch 日志 |

## 第二终端接入

主终端启动后，在另一终端执行：

```bash
source /tmp/cr5_spray_simulation.env

# 查看话题
rostopic list

# 查看 TF
rosrun tf tf_echo world calibration_target_frame

# 查看相机图像
rosrun image_view image_view image:=/cam_front_left/camera/color/image_raw
```

## 停止仿真

在主终端按 Ctrl+C，脚本自动执行清理：
- 停止 roslaunch 进程组
- 停止本脚本创建的 roscore
- 删除 `/tmp/cr5_spray_simulation.env`
- 释放单实例锁

## 故障排查

### 端口被占用

如果默认端口 11311/11345 被占用，脚本会自动 fallback 到可用端口。
如果持续出现端口递增，说明上次会话的 roscore 未被清理。
重启后首次启动应回到默认端口。

### 相机模型未生成

检查 Gazebo 日志中是否有 "look-at failed" 或 "xacro failed"。
确认 `config/scene_v31.yaml` 存在且格式正确。

### 标定贴图显示为黄黑占位图

确认 `models/calibration_target/materials/textures/` 下 PNG 文件存在。
确认 `calibration_target.material` 中 texture 名与 PNG 文件名一致。
