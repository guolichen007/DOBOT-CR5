# 项目架构与代码说明

## 项目简介

DOBOT-CR5 多相机标定与三维重建项目。基于 ROS Noetic + Gazebo Classic 11 仿真环境，
使用三台固定 D455-like RGB-D 相机对标定目标进行多视角采集、外参标定和三维重建。

## ROS 包结构（7 个包）

| 包 | 职责 |
|----|------|
| `dobot_bringup` | CR5 实机驱动：TCP 通信、运动控制、状态消息 |
| `dobot_description` | CR5 机械臂 URDF/Xacro 描述（上游供应商包） |
| `cr5_moveit` | MoveIt 运动规划配置 |
| `cr5_spray_sim` | Gazebo 仿真环境：场景、模型、相机、启动脚本 |
| `cr5_spray_perception` | 感知处理：相机采集、标定验证、外参估计 |
| `realsense_gazebo_description` | D455-like 相机 Gazebo 描述（Xacro + 配置） |
| `realsense_gazebo_plugin` | RealSense Gazebo 插件（上游包） |

## 包间关系

```
dobot_description ──→ cr5_moveit (MoveIt 需要 URDF)
realsense_gazebo_description ──→ cr5_spray_sim (相机 Xacro)
realsense_gazebo_plugin ──→ cr5_spray_sim (Gazebo 传感器插件)
dobot_bringup ←── 实机部署时独立运行
cr5_spray_perception ←── 标定数据采集和处理
```

## 主要 TF 树

```
world → base_link → Link1..Link6 (CR5 机械臂)
world → object_frame → calibration_target_frame
  ├── calibration_target_front_frame  (ChArUco 8×6)
  ├── calibration_target_left_frame   (ChArUco 6×5)
  ├── calibration_target_right_frame  (AprilTag 2×2)
  ├── calibration_target_top_frame    (AprilTag 1×1)
  └── calibration_target_back_frame   (ChArUco 8×6)
```

## 分支策略

| 分支 | 职责 |
|------|------|
| `main` | 已验证、可构建、可运行的稳定工程基线 |
| `develop/multi-camera-calibration` | 三相机外参与三维重建开发 |

短期任务分支命名: `feature/<主题>` / `fix/<主题>` / `chore/<主题>`。
完成后必须合并到 develop 或 main，然后删除远端分支。

禁止直接在 `main` 上做大改动。
