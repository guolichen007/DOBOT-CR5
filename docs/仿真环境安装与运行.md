# 仿真环境安装与运行

## 系统要求

- Ubuntu 20.04 LTS
- ROS Noetic (完整桌面版)
- Gazebo Classic 11
- Python 3.8+

## 依赖安装

```bash
# ROS Noetic 安装 (如未安装)
# 参考: http://wiki.ros.org/noetic/Installation/Ubuntu

# 系统依赖
sudo apt install -y \
  ros-noetic-gazebo-ros-pkgs \
  ros-noetic-gazebo-ros-control \
  ros-noetic-moveit \
  ros-noetic-tf2-ros \
  ros-noetic-cv-bridge \
  ros-noetic-image-transport \
  ros-noetic-camera-info-manager \
  python3-catkin-tools python3-numpy python3-yaml python3-opencv

# rosdep 初始化 (如未执行)
sudo rosdep init
rosdep update
```

## 克隆与构建

```bash
cd ~
git clone git@github.com:guolichen007/DOBOT-CR5.git cr5_ros1_ws
cd cr5_ros1_ws
rosdep install --from-paths src --ignore-src -r -y
catkin_make
source devel/setup.bash
```

## 启动仿真

```bash
# GUI 模式 (推荐调试)
bash src/cr5_spray_sim/scripts/run_simulation.sh \
  --gui --object=calibration_target --profile=quality

# Headless 模式 (批量测试/CI)
bash src/cr5_spray_sim/scripts/run_simulation.sh \
  --headless --object=calibration_target --profile=quality --strict
```

### 命令行参数

| 参数 | 说明 |
|------|------|
| `--gui` | 显示 Gazebo GUI |
| `--headless` | 无头模式 (默认) |
| `--object` | 工作对象: `calibration_target`, `motor_housing_cylinder`, `rectangular_housing` |
| `--profile` | 相机分辨率: `vm` (低分辨率), `quality` (标准分辨率) |
| `--strict` | 任何检查失败立即退出 |
| `--isolated` | 独立模式 (随机端口，用于批量测试) |
| `--no-spray-sim` | 不启动喷涂控制 |

## 第二终端接入

仿真启动后，在另一终端：

```bash
source /tmp/cr5_spray_simulation.env

# 查看话题
rostopic list

# 查看 TF
rosrun tf tf_echo world calibration_target_frame

# 查看相机图像
rosrun image_view image_view image:=/cam_front_left/color/image_raw
```

# 检查所有三台相机 depth 话题

## 日志目录

仿真日志存储在 `/tmp/cr5_spray_sim_<session_id>/`，包含:
- 完整 roslaunch 输出
- diagnostics 快照
- 环境变量文件

## 停止与清理

主终端 Ctrl+C 自动执行：
- 停止 roslaunch 进程组
- 停止 roscore
- 删除 `/tmp/cr5_spray_simulation.env`
- 释放端口

## 常见故障

### 端口被占用

脚本会自动 fallback 到可用端口。如果持续递增，说明上次 roscore 未正常退出。
重启后首次启动应回到默认端口。

### 相机模型未生成

检查日志中是否有 "look-at failed" 或 "xacro failed"。
确认 `config/simulation_scene.yaml` 存在且格式正确。

### 标定贴图异常

确认 `models/calibration_target/materials/textures/` 下五张 PNG 存在。
确认 `calibration_target.material` 中 texture 名与 PNG 文件名一致。

### 进程残留

```bash
pkill -9 gzserver gzclient roscore rosmaster 2>/dev/null
```
